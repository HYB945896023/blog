网络知识 #

1. 请求类型大全 #

  该属性作用就是告诉客户端、服务端双端 所传输的媒体数据 是什么类型的，以便于双方使用对应的解码类型去解析数据。 也被称为MIME类型

  1. 普通媒体模式：
字段格式
text/htmlHTML格式
text/xmlXML格式
image/gifgif图片格式
image/jpegjpg图片格式
image/pngpng图片格式

  1. application媒体类型

    1. application/x-www-form-urlencoded： 浏览器的原生form表单，传输以key1=val1&key2=val2 的形势进行传输
    2. application/pdf
    3. application/xml
    4. application/json
    5. application/msword : word文档的格式
    6. application/octet-stream： 二进制流文件，常见的文件下载
    7. application/atom+xml： Atom XML聚合格式
    8. application/xhtml+xml ： XHTML格式

  2. Multipart/form-data: 媒体上次文件类型 Form标签上的enctype="multipart/form-data" 使用boundary进行单元分隔.

2. https与http的不同 #

  • http：不加密传输，有很大的安全性问题
  • https默认端口为443
  • https在http基础上增加了SSL/TSL协议，负责网络连接的加密， https建立安全的连接通道，服务端发送一个网站的ssl证书信息给客户端 使用非对称与对称加密.
  • 对称加密：双方都有一套统一的算法进行加密，也就拥有相同的私钥
  • 非对称加密：核心理念：公钥加密的数据只能使用私钥解密 服务端拥有一个公钥与一个私钥 服务端发送公钥给客户端，客户端使用公钥进行加密，而解密只能由服务端的私钥进行解密。

https具体步骤说明：

  1. 客户端发送第一个随机数（TLS版本、加密套件，随机数1），服务端保存
  2. 服务端确认客户端发送的信息，发送第二个随机数（TLS版本、加密套件，随机数2）
  3. 服务端将 证书 与 公钥 发给客户端，
  4. 客户端生成第三个随机数（预主密钥），将预主密钥使用公钥进行加密，发送到服务端 服务端使用私钥 解密得到预主密钥
  5. 客户端使用 第一随机数+第二随机数 + 预主密钥 = 会话密钥 服务端 第一随机数+第二随机数 + 预主密钥 = 会话密钥
  6. 此时双方使用会话密钥 进行 对称加密通话。

3. http1.0与http1.1的不同 #

  1. 1.1增加了长连接，一次连接可以发送多个http请求，而1.0则需要配置keep-alive才可以建立长连接
  2. 增加一些错误码 (409（Conflict）表示请求的资源与资源的当前状态发生冲突；410（Gone）表示服务器上的某个资源被永久性的删除)
  3. 增加了缓存相关的header (Entity tag，If-Unmodified-Since, If-Match, If-None-Match等更多可供选择的缓存头来控制缓存策略。)
  4. 增加HOST域：1.0认为一台物理服务器只有一个主机名，而实际上现在一台服务器可开启多个虚拟主机，指向同一个ip地址，1.1增加了host域
  5. 支持服务器只传输一部分内容，例如只传送header，不传body

4. http1.1与http2.0的不同 #

  1. 2.0采用了多路复用技术，一个连接并发处理多个请求，请求数量多了1.1几个量级
  2. 2.0支持头部压缩：1.1只压缩请求体，2.0可以压缩header，在面对请求数量巨大的时候有效避免了资源的浪费
  3. 服务器推送：它允许服务端推送资源给浏览器，在浏览器明确地请求之前

5. http的缓存策略 #

协商缓存:

  • last-modified 发送请求查看最后一次请求时间与服务端资源更新时间是否一致,不一致则服务端返回最新资源.
  • ETag etag为资源的唯一标识,发送请求查看该资源的标识与服务端资源当前资源的标识是否一致,不一致则服务端返回最新资源.

计算机网络知识： #

  • 带宽: 一秒内能达到的最大速度，也就是速度的理论极值
  • 吞吐量: 单位时间内通过某接口，信道的数据量

数据传输使用电气特性来传输，10-15表示二进制0，-10- -15表示1

  • 光纤:使用光脉冲来传输，与电平信号进行互相转换。

  • 传输会有损耗，所以我们需要每隔一段距离来还原增强信号，这个东西就是中继器

  • 以太网:局域网技术 无连接:发送方与接收方不会建立连接。 不可靠:接收方无法向发送方确实 数据经过网卡变成传输帧

  • 无线局域网: 利用AP通信基站来通信，实行半双工通信模式

  • 网段:ip地址+子网掩码

  • 我们通过路由NAT转换与外界网络进行通信，一般一个c类地址会划分为多个子网，外部网络请求通过子网掩码来寻找目标子网。 ARP协议，就是用来寻找目标mac地址进行通信，每个主机都有一个ARP列表，每次发送请求前先检查自身的ARP列表是否有目标源ip与mac地址，没有就发送ARP请求寻找目的主机，网络中每个收到ARP请求会先去查看自身ip等信息是否符合，符合将信息插入自身ARP列表（包含了源主机的信息，然后返回响应。

  • ICMP协议，用来校验是否有丢包以及正常传输，ping就是基于该协议。

  • 端口用来监听一个应用程序，网络请求需要确定发送给哪个应用程序，所以一个端口只能有一个应用程序。

  • udp没有三握四挥，减少了开销，以及数据不会分片（数据链路层的每一帧是有大小限制的），没有拥塞控制，无论网络情况照常发送自己的数据
