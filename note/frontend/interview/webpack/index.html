webpack 原理解析: #

webpack 是一个打包工具,是前端模块化、工程化的始组，它可以让繁多 html、css、js 以模块的形式进行打包，打包成一个或者多个文件，使得项目的结构清晰，让依赖与模块的变量之间不会产生混乱，可以减少文件请求的次数,本质是commonjs模块化,使用 require 引入其他模块，将多个文件按模块打包，打包成一个自动执行函数。

  • 可解析 json、vue、jsx、scss、less 等其他文件，以及高版本的 js(ES6~)代码，解析成可供浏览器识别的原始三种文件，该点是以 Loader 的形式实现。
  • 由 loader 解析完后，通过 plugins 进一步优化项目，例如代码压缩，图片压缩，树摇等等强大的功能。

Vue 中的 webpack #

Vue2 项目正是依赖 webpack 打包运行，前提是使用 vueCli 创建的项目，vueCli本质上一个命令行处理工具，根据用户的选择:

  • 拉取模板
  • 合并配置 webpack 的配置
  • 安装各种依赖，并且配置好命令文件 创建好项目后，npm run build 命令启动项目，该命令会指向 node_module 中的可执行脚本文件 webpack，具体步骤如下:
  1. npm run build
  2. 找到 node_module 里面的 bin 目录下的可执行文件 vue-cli-service,按照执行依赖文件一个个执行，最终执行 webpack 函数。
  3. webpack 函数导出一个compiler实例对象，调用compiler.run开始项目执行。

Webpack 解析 #

webpack 函数接受配置文件，将配置文件传给 compiler 类,并且实例化该类，并将该实例作为plugins的参数,让plugins去订阅各个生命周期钩子，在 webpack 执行到这些钩子时，会自动去调用这些插件提供的方法，这就是插件的基本原理。

js
// 主要入口函数
function webpack(webpackOptions) {
  const compiler = new Compiler(webpackOptions);
  // 解构出配置中的插件
  const { plugins } = webpackOptions;
  // 调用每个插件上的apply方法，实现插件订阅绑定事件操作
  for (const plugin of plugins) {
    // 本质就是调用compiler实例上绑定的hooks，调用tap方法进行绑定回调
    plugin.apply(compiler);
  }
  return compiler;
}
module.exports = webpack;

1. Compiler 类 #

   compiler初始化先注册各种插件，并且在 run 方法中规定好各个钩子的触发时机，把这套逻辑封装成回调函数传入Compilation的build方法中，待内部执行完毕则返回执行 call 触发钩子，将打包完成的代码写入磁盘.

webpack的热更新原理: #

wds(服务端)与浏览器中间维护了一个websoket，当代码发生变化，wds会推送更新后的文件hash值到浏览器，浏览器拿到后分析与之前的hash是否一致，若不同则发送ajax获取文件列表与最近hash等文件修改内容，客户端再借助这些信息向wds发送jsonp获取chunk增量更新，具体更新使用hotmoduleplugin插件。

tapable #

tapble 源码 tapble 是一个优秀的发布订阅库，通过 tap 去订阅函数，call 的时候会去执行所有 tap 过的函数，webpack的插件机制就是依靠 tapble 实现.

js
// Compiler类，每次rnpm run build就会创建一个Compiler
class Compiler {
  constructor(webpackOptions) {
    // 存储配置信息
    this.options = webpackOptions;
    // 初始化Compiler挂载钩子
    this.hooks = {
      run: new SyncHook(),
      done: new SyncHook(),
    };
  }
  // 真正做编译的函数
  compile(callback) {
    // 每一次文件更新都会创建一个compilation对象，是webpack打包的核心
    let compilation = new Compilation(this.options);
    // 执行build方法开始编译
    compilation.build(callback);
  }
  run(callback) {
    // 去触发之前绑定好的run钩子，所有绑定了run钩子的回调都会被执行
    this.hooks.run.call();
    // 定义编译完成的回调，里面同样去触发done钩子
    const onCompiler = (err, stats, fileDependencies) => {
      // 编译完成根据配置信息，将内容写入磁盘中
      for (let filename in stats.assets) {
        // 获取输出文件路径
        let filePath = path.join(this.options.output.path, filename);
        fs.writeFileSync(filePath, stats.assets[filename], "utf-8");
      }
      // watch监视模式
      fileDependencies.forEach((fileDependencie) => {
        fs.watch(fileDependencie, () => this.compile(onCompiled));
      });
      callback(err, {
        toJson: () => stats,
      });
      this.hooks.done.call();
    };
    // 调用编译函数，正式开始编译
    this.compile(onCompiler);
    // 编译完成执行回调
    callback();
  }
}

2. Compilation 类 #

每次文件更新都会创建一个compilation对象,以模块化划分代码文件,规定每个模块的结构,以对象的形式规定模块 chunk,详细对象定义规则见代码与示例，打包流程:

  1. 从入口文件出发，递归查找依赖的文件，打包成一个文件，如果有多个入口则打包成多个 chunk。
  2. 查找过程：判断入口个数，然后从入口模块开始逐步生成模块对象，生成的模块对象组合成 chunk，根据chunk内容生成assets文件,assets就是文件代码内容,最终写入代码到 dist(出口文件)中
  3. 生成模块： 由buildModule方法执行,获取每个模块的 ID,也就是与入口文件的相对路径,这个过程会递归执行，直到入口文件的依赖文件全部找到，查找方法：将代码转化成 AST 抽象语法树，查找 require 所引入的模块，查找到则加入dependencies当中,后续会遍历dependencies，通过模块 ID 找到所依赖的文件，继续递归执行上述操作。

  _source: "", // 模块的代码信息

// Compilation类
class Compilation {
  constructor(webpackOptions) {
    this.options = webpackOptions;
    this.modules = []; // 本次编译所有生成的模块
    this.chunks = []; // 本次编译所有的代码块，将入口文件和其依赖打包一起
    this.assets = {}; // 本次编译产生的资源文件
    this.fileDependencies = []; // 本次打包所涉及的所有文件，为watch模式监听变化以便重新打包
  }
  // 这里开始编译流程
  // 首先需要寻找入口文件
  build(callback) {
    // 1.首先需要寻找入口文件,并找到其依赖
    let entry = {};
    if (typeof this.options.entry === "string") {
      // 单入口要转化为main
      entry.main = this.options.entry;
    } else {
      // 多入口
      entry = this.options.entry;
    }
    // 从入口文件出发，使用loader来转化文件
    for (let entryName in entry) {
      //path.posix为了解决不同操作系统的路径分隔符,这里拿到的就是入口文件的绝对路径
      let entryFilePath = path.posix.join(baseDir, entry[entryName]);
      // 将绝对路径加入到文件依赖数组
      this.fileDependencies.push(entryFilePath);
      // 获取入口模块的module对象，里面存放了模块的详细信息，包括 路径，依赖模块路径，源代码
      let entryModule = this.buildModule(entryName, entryFilePath);
      // 进行代码块组装，入口模块与其依赖的所有模块
      let chunk = {
        name: entryName, // 代码块名称
        entryModule, // 模块对象
        modules: this.modules.filter((item) => item.names.includes(entryName)), // 找出属于该代码块的模块
      };
      this.chunks.push(chunk);
    }
    this.chunks.forEach((chunk) => {
      console.log("---------------------------------------");
      // 获取出口文件名称,[name]:则为chunk名称
      let filename = this.options.output.filename.replace("[name]", chunk.name);
      // 获取代码块的 执行代码
      this.assets[filename] = getSource(chunk);
    });
    // 编译成功，执行回调
    callback(
      null,
      {
        chunks: this.chunks,
        modules: this.modules,
        assets: this.assets,
      },
      this.fileDependencies
    );
  }
  // 读取模块内容，返回一个完整的module对象
  buildModule(name, modulePath) {
    let sourceCode = fs.readFileSync(modulePath, "utf8");
    // 模块的id就是相对于根目录的相对路径，path.relative就是获取相对路径的方法（./src/index.js）
    let moduleId = "./" + path.posix.relative(baseDir, modulePath);
    // 本模块的信息对象
    let module = {
      id: moduleId, // 模块的唯一标识
      names: [name], // 该模块的属于哪个代码块
      dependencies: [], // 该模块依赖于这些模块
      _source: "", // 模块的代码信息
    };
    // 读取loader配置，将源代码进行翻译与替换
    let loaders = [];
    let { rules = [] } = this.options.module;
    rules.forEach((rule) => {
      let { test } = rule;
      // 匹配文件正则
      if (modulePath.match(test)) {
        // use就是loader函数
        loaders.push(...rule.use);
      }
    });
    // 自右向左对模块进行转译，reduceRight从尾到头遍历累加
    sourceCode = loaders.reduceRight((code, loader) => {
      return loader(code);
    }, sourceCode);

    // 将代码转化为AST
    let ast = parser.parse(sourceCode, { sourceType: "module" });
    // 在AST中修改代码，使用正则直接匹配替换会容易出错
    traverse(ast, {
      // 本模块：当前读取的模块  依赖模块：本模块中require的对象
      CallExpression: (nodePath) => {
        const { node } = nodePath;
        if (node.callee.name === "require") {
          let depModuleName = node.arguments[0].value; // 获取依赖的模块,require所引入的模块
          let dirname = path.posix.dirname(modulePath); // 获取当前正在编译的模所在的目录
          let depModulePath = path.posix.join(dirname, depModuleName); // 获取到依赖模块的绝对路径
          let extensions = this.options.resolve?.extensions || [".js"]; // 读取extension配置
          depModulePath = tryExtensions(depModulePath, extensions); // 尝试添加后缀，找到真实的文件
          // 将所依赖的模块的路径添加到该 chunk里面
          this.fileDependencies.push(depModulePath);
          // 生成依赖模块的 模块ID
          let depModuleId = "./" + path.posix.relative(baseDir, depModulePath);
          // 修改语法结构，将require('dep') => require('./src/node_module/dep.js')
          node.arguments = [types.stringLiteral(depModuleId)];
          // 将依赖模块的信息push到 本模块信息 的dependencies属性中
          module.dependencies.push({ depModuleId, depModulePath });
        }
      },
    });
    // 将处理的ast转化为正常代码
    let { code } = generator(ast);
    module._source = code;
    // 遍历依赖模块，获取依赖模块的 模块对象（对 `module 对象`中的 `dependencies` 进行递归执行 `buildModule` ）
    module.dependencies.forEach(({ depModuleId, depModulePath }) => {
      // 考虑多入口打包，防止重复打包，最终返回的module对象
      let exitModule = this.modules.find((item) => item.id === depModuleId);
      // 已经存在则无需编译，将此代码块名称添加到对应模块的names
      if (exitModule) {
        // 指明其属于哪个代码块，chunk
        exitModule.names.push(name);
      } else {
        // 递归获取依赖模块的 module对象
        let depModule = this.buildModule(name, depModulePath);
        // console.log(depModule);
        this.modules.push(depModule);
      }
    });
    // 返回模块对象
    return module;
  }
}

// 转化路径中的 \ 为 /
function toUnixPath(filePath) {
  return filePath.replace(/\\/g, "/");
}
// 获取基本路径，process.cwd就是获取指令执行时所处的目录路径
const baseDir = toUnixPath(process.cwd());

// 尝试获取文件真实路径
function tryExtensions(modulePath, extensions) {
  // console.log(modulePath);
  // 存在该文件则直接返回，这是引入时带后缀的情况
  if (fs.existsSync(modulePath)) {
    return modulePath;
  }
  // 遍历extension尝试添加后缀
  for (let i = 0; i < extensions?.length; i++) {
    let filePath = modulePath + extensions[i];
    if (fs.existsSync(filePath)) {
      return filePath;
    }
  }
  throw new Error(`无法找到${modulePath}`);
}
// 生成运行时的代码
function getSource(chunk) {
  return `
    (()=>{
        var modules = {
            ${chunk.modules.map(
              (module) =>
                `"${module.id}":(module)=>{
                    ${module._source}
                }`
            )}
        };
        var cache = {};
        function require(moduleId){
            var cacheModule = cache[moduleId];
            if(cacheModule !== undefined){
                return cacheModule.exports
            }
            var module = (cache[moduleId] = {
                exports:{}
            })
            modules[moduleId](module,module.exports,require)
            return module.exports
        }
        var exports = {}
        ${chunk.entryModule._source}
    })()
    `;
}
