虚拟dom原理 & Diff算法原理 #

虚拟dom算法,将dom转化成js对象,也就是使用js对象的形式来表示实际的dom树。

为什么需要虚拟dom，直接使用dom不好吗？ #

了解浏览器的渲染机制就能知道，频繁的操作dom会引起浏览器不断的进行回流、重绘，对于页面性能有着很大的损耗，因此我们应****尽量减少dom的操作次数**，使用js对象来存储dom信息,增删改查都在js对象上面，直到比对完毕后才插入dom树上,极大提高了页面性能与用户体验。

虚拟dom #

本文以snabdom为参照,简版虚拟dom,下面就以这个为默认版本,虚拟dom是一个对象，上面有三个属性：sel:描述dom节点的类型，prop存储dom的属性,第三个属性 有两种情况： 文本内容 子节点的数组，如下所示。

1. h函数 #

使用h函数生成我们需要的虚拟dom,根据入参生成不同的虚拟dom对象

js
import vnode from "./vnode";
// 低配h函数
/* 
传入的只有以下三种情况
1.h('div',{},'text')
2.h('div',{},[])
3.h('div',{},h())
 */
export default function (sel, data, c) {
  // 检查参数个数
  if (arguments.length != 3) {
    throw new Error("我这是低配版，你别乱传参数给我，懂？");
  }
  // 为第一种情况，直接添加为vnode即可
  else if (typeof c == "string" || typeof c == "number") {
    return vnode(sel, data, undefined, c, undefined);
  }
  // 为第二种情况,c参数是数组
  else if (Array.isArray(c)) {
    // 新建一个子数组存放该节点的子节点
    let children = [];
    for (let i = 0; i < c.length; i++) {
      // 數組裏面一定要是h函数
      if (typeof c[i] == "object" && c[i].hasOwnProperty("sel")) {
        children.push(c[i]);
      } else {
        throw new Error("数组中存在不是h函数的项");
      }
    }
    // 返回带子节点数组的 虚拟节点,最后两个参数：text elm
    return vnode(sel, data, children, undefined, undefined);
  }
  // 第三种情况,c参数是h函数
  else if (typeof c == "object" && c.hasOwnProperty("sel")) {
    // 直接把c放在children中,不需要执行，在使用时h()就会自动去调用它
    return vnode(sel, data, [c], undefined, undefined, undefined);
  } else {
    throw new Error("传入的第三个参数有问题");
  }
}

2. 虚拟dom转化成真实dom & diff算法 #

虚拟dom转化为真实dom是一个复杂的过程,patch函数负责实现这个过程, patch函数会接受两个节点patch(vnode1,vnode2)，这两个节点可以是真的dom/虚拟dom,最终会将最新的dom更新挂载到dom树上,且在比较两个dom的时候就会采用diff算法。

什么是diff算法？ #

  diff算法也叫做最小量更新,虚拟dom的消耗就在于在更新dom前的比对,若前后不变那么就可以保留原节点,但倘若不做任何优化,那么每个旧节点都要与新节点去比较一次，时间复杂度为：n^3,是一个相当大的消耗,diff算法就是优化这个过程的。 diff算法遵循三个优化的原则：

  1. key是唯一标识，不加Key统一在最后插入节点，并且会去逐一改变我们的原始节点。
  2. 只有同一个虚拟节点才会去做精细化比较，例如前后都是ol，不然会直接拆卸插入新的。
  3. 只会进行同层的比较，例如：原来ol>li 现在ol>ul>li,这种情况下不会去进行精细化比较，而是进行直接拆卸。 依据这三个原则可极大提升dom比对的效率。

patch函数 #

Patch的步骤：

  1. 判断旧节点是否为虚拟节点 是则跳到第二步 否则转化为虚拟节点 跳到第二步。（就是保证比较的两个节点都为虚拟节点）
  2. 判断旧节点与新节点是否为同一节点，是则进行精细化比较，否则将旧节点拆卸,并将新节点插入。 精细化比较流程： 判断是否为统一对象：是>什么都不做 否：判断新节点是否有text属性，是：> 替换掉旧节点的属性 否：新节点是否有children：是：> 旧节点没有children,将新节点children直接赋值 (在我们的简约版案例中children与text是互斥的) 新旧节点都有children(复杂情况)：
    1. 新前旧前
    2. 新后旧后
    3. 新后旧前
    4. 新前旧后 四个指针依次去对比，前指针命中则往下走，后指针命中往上走 循环条件是： while( 新前<=新后 && 旧前<=旧后 )
    5. 四种都没有命中：会循环去旧节点中查找,将旧节点中的该项设为undefined,并把该节点放在旧前指针之前(以新dom为准去修改旧dom),这里会放一个缓存，每次从一个对象中去查找 没有找到该节点，则创建新节点
    6. 判断是否还有剩余节点，继续做如下处理： 旧节点先循环完毕则，新节点插入到旧节点后面。 新节点先循环完毕，将剩余的旧节点全部删除

patch函数源码： #

js
import vnode from './vnode'
import createElement from './createElement';
import patchVnode from './patchVnode'
// 该函数将真正的虚拟dom上树
export default function patch(oldVnode,newVnode){
    // 第一步判断旧节点是否为虚拟节点
    if(oldVnode.sel == '' || oldVnode.sel == undefined){
        // 说明旧节点为dom节点,old作为elm传入
       oldVnode =  vnode(oldVnode.tagName.toLowerCase(),{},[],undefined,oldVnode)
    }
    // 第二部判断旧节点与新节点是否为同一个节点
    if(oldVnode.sel == newVnode.sel && oldVnode.key == newVnode.key){
        // 同个节点的精细化比较
        patchVnode(oldVnode,newVnode)
    }
    // 说明不是同一个节点，需要去创建新的插上去,删除旧的
    else{
        // 调用createElement，将new节点变成真的dom
       let newDom = createElement(newVnode)
       if(newDom){
        // 插入新节点的位置是在标杆节点（旧节点）之前
        // insertBefore是往父元素差节点，插入到标杆节点的上一级，第二个参数是插在该节点之前
        // 上树
          oldVnode.elm.parentNode.insertBefore(newDom,oldVnode.elm)
       }
         // 删除旧节点
         oldVnode.elm.parentNode.removeChild(oldVnode.elm)
    }
}

updateChildren源码（精细化比较，diff算法核心）: #

js
import patchVnode from './patchVnode'
import createElement from './createElement'
// 判断两个虚拟节点是否为同一节点
function checkSameVnode(a,b){
    return a.sel == b.sel && a.key == b.key
}
// 根据四个指针去操作
export default function updateChildren(parentElm,oldCh,newCh){
    // 旧前指针
    let oldStartIdx = 0
    // 新前指针
    let newStartIdx = 0
    // 旧后指针
    let oldEndIdx = oldCh.length - 1
    // 新后指针
    let newEndIdx = newCh.length - 1

    // 旧前节点
    let oldStartVnode = oldCh[0]
    // 新前指针
    let newStartVnode = newCh[0]
    // 旧后指针
    let oldEndVnode = oldCh[oldEndIdx]
    // 新后指针
    let newEndVnode = newCh[newEndIdx]
    // 用于四种都没命中，将旧节点的值放进该对象中，用于新节点的寻找匹配
    let KeyMap = null
    // 当后指针移动到前指针前面时，循环结束
    while(oldStartIdx <= oldEndIdx && newStartIdx <= newEndIdx ){
        // 旧节点循环查找中，把找到的节点设置为undefined，所以进入前需要判断
        if(oldStartVnode == null){
            oldStartVnode = oldCh[++oldStartIdx]
        }
        else if(oldEndVnode == null){
            oldEndVnode = oldCh[--oldEndIdx]  
        }
        // 新前与旧前命中
       else if(checkSameVnode(oldStartVnode,newStartVnode)){
            console.log('命中了1新前旧前');
            // 递归patchVnode进行两子节点的精细化比较，更新旧节点内容
            patchVnode(oldStartVnode,newStartVnode)
            // 指针下移改变匹配到的子节点
            oldStartVnode = oldCh[++oldStartIdx]
            newStartVnode = newCh[++newStartIdx]
        }
        // 命中 新后旧后
        else if(checkSameVnode(oldEndVnode,newEndVnode)){
            console.log('命中了2新后旧后');
            patchVnode(oldEndVnode,newEndVnode)
            // 指针上移改变匹配到的子节点
            oldEndVnode = oldCh[--oldEndIdx]
            newEndVnode = newCh[--newEndIdx]
        }
        // 命中 新后旧前
        else if(checkSameVnode(oldStartVnode,newEndVnode)){
            console.log('命中了3新后旧前');
            // 更新节点内容
            patchVnode(oldStartVnode,newEndVnode)
            // 把更新后的旧前节点移动到旧后的后面,新的放在最后，所以旧的也得放在最后
            parentElm.insertBefore(oldStartVnode.elm,oldEndVnode.elm.nextSibling)
            // 移动指针
            oldStartVnode = oldCh[++oldStartIdx]
            newEndVnode = newCh[--newEndIdx]
        }
        // 命中 新前旧后
        else if(checkSameVnode(newStartVnode,oldEndVnode)){[]
            console.log('命中了4新前旧后');
            // 更新节点内容
            patchVnode(oldEndVnode,newStartVnode)
            // 把更新后的旧后节点移动到旧前的前面
            parentElm.insertBefore(oldEndVnode.elm,oldStartVnode.elm)
            // 移动指针
            newStartVnode = newCh[++newStartIdx]
            oldEndVnode = oldCh[--oldEndIdx]
        }
        // 都没有命中，去旧节点循环查找，然后将节点放到旧前指针前面
        else{
            console.log('四个规则都没有命中');
            // 把key放进对象中
            if(!KeyMap){
                 KeyMap = {}
                // 去遍历旧节点的所有key,并存进这个对象中，使用对象[属性]查找索引
                for (let i = oldStartIdx; i <= oldEndIdx; i++) {
                    const key = oldCh[i]?.key
                    if( key!= undefined ){
                        // 将key存进去
                        KeyMap[key] = i
                    }        
                }
            }  
            // 去寻找newStartIdx在oldCh中的哪一项
            const oldIdx = KeyMap[newStartVnode.key]

            // 找到了说明是旧节点是存在该项的，需要移动
            if(oldIdx != undefined){
                const vnode = oldCh[oldIdx]
                console.log('在旧节点中遍历找到了'+vnode.key+'将其移动到最前面');
                patchVnode(vnode,newStartVnode)
                // 将该节点设置为undefine
                oldCh[oldIdx] = undefined
                // 移动此节点/插入到 旧前的前面
                parentElm.insertBefore(vnode.elm,oldStartVnode.elm)
            }
            // 没找到说明是全新的项，直接新建后加入
            else{
                console.log('在旧节点中遍历中未找到，是新节点需要插入' + newStartVnode.key);
                // 插入到旧前
                parentElm.insertBefore(createElement(newStartVnode),oldStartVnode.elm)
            }   
            // 匹配到后将新前指针往下移
            newStartVnode = newCh[++newStartIdx]
        }
    }  
    // 循环结束，判断新节点中是否有剩余的，剩余则需要插入
    if(newStartVnode <= newEndVnode){
        // 将剩余的节点插入到新节点的尾部中,insertBefore识别null
        // 理解：循环完毕后旧节点的内容已经全部找到且是以新节点的顺序，所以此处插入应该以新节点的新后后一项为基准
        let before = newCh[newEndIdx+1] == null? null : newCh[newEndIdx+1].elm
        for (let i = newStartIdx; i <= newEndIdx; i++) {
            console.log('旧节点循环结束，需要插入新节点'+newCh[i].key);
            // 前面都命中修改完毕后，将新节点上树
            parentElm.insertBefore(createElement(newCh[i]),before)    
        }
    }
    // 循环结束，判断旧节点是否有剩余，剩余则需要删除
    else if( oldStartIdx <= oldEndIdx){
        console.log('新节点循环结束，需要删除旧节点');
        // 删除旧节点开始与结束间的内容
        for (let i = oldStartIdx; i <= oldEndIdx; i++) {
            if(oldCh[i]){
                parentElm.removeChild(oldCh[i].elm)   
            }       
        }
    }
}

createElement源码 #

// 将新插入的虚拟节点转化为真的dom
export default function createTrueElement(vnode){
    // 根据虚拟节点创建dom节点
    let domNode = document.createElement(vnode.sel)
    // 判断该节点是否有子节点，这里判断无子节点
    if(vnode.text != '' &&(vnode.children == undefined || vnode.children.length == 0)){
        // 通过判断的节点就是纯文本节点，无子节点
        domNode.innerText = vnode.text
    }
    // 判断是否有子节点，这里判断有子节点
    else if(Array.isArray(vnode.children) && vnode.children.length > 0 ){
        for (let i = 0; i < vnode.children.length; i+){   
            // 这里递归去去获取子数组里面的dom, 纯文本节点为递归出口
                let chDOM = createTrueElement(vnode.children[i])
                // 将子数组加到树上
                domNode.append(chDOM)
        }
    }
    vnode.elm = domNode
    // 最终返回的是dom节点
    return vnode.elm
}
